> Companion to `FUTURES_ENABLEMENT_DEEP_DIVE_PROMPTS.md` and the EQUITY enablement pack.
> Each prompt is a self-contained, anchor-verified work order to **design and implement the optimal
> solution** for one slice of options-trading enablement, in the same house format
> (Mission / Prerequisites / Global constraints / In scope / Out of scope / Verified anchors /
> Tests / Validation commands / Self-audit & NO-GO), ending with the mandated audit-and-NO-GO paragraph.
>
> **Current baseline (what these prompts change):** options are ingested and turned into *signals on the
> underlying* (IV rank, skew, term structure, GEX, flow), but the system **cannot trade options** ‚Äî live
> orders are hard-blocked (`LIVE_OPTIONS_BROKER_ADAPTERS = frozenset()`), `OPTIONS_INSTRUMENTS_MODE`
> defaults to `shadow`, `broker_sim` treats options 1:1 like shares, there is no OPTION asset class, no
> greeks-based portfolio risk, and no option-lifecycle handling.
