# config_env_allowlist.txt
#
# Frozen legacy backlog of environment variables that ARE read in code
# (engine/, services/, routes/, tools/, boot/, ops/, scripts/, and root *.py via
# os.getenv / os.environ.get / os.environ[...]) but are NOT yet documented in
# .env.example or docs/REFERENCE_CONFIGURATION_GLOSSARY.md.
#
# Purpose: the env-var coverage gate in tools/validate_docs.py fails the build when
# a NEW undocumented env var is introduced. This allowlist tolerates the pre-existing
# backlog so the gate can be enforced immediately without a mass-documentation effort.
# It does NOT bless these as well-documented -- it freezes the gap so it cannot grow.
#
# How to shrink it: document a variable in .env.example or the configuration glossary
# (docs/REFERENCE_CONFIGURATION_GLOSSARY.md), then DELETE its line here and re-run
# `python tools/validate_docs.py`. Never ADD a new variable here to dodge the gate --
# document it instead.
#
# One UPPER_SNAKE env var per line, sorted. Lines starting with # are comments.
