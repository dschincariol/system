# Complete, reproducible dependency set for the OPT-IN AMD ROCm profile.
# Target: Strix Halo / gfx1151 (Ryzen AI MAX) on the repo's CPython 3.12 baseline.
# Validated on host `bart`, 2026-06-21 with AMD's ROCm 7.2.4 PyTorch image.
#
# WHY THIS FILE EXISTS
#   `requirements-amd-rocm.txt` is the authoritative ROCm wheel overlay used by
#   compose. This file expands it with the base runtime deps for operators who
#   want a standalone host venv instead of the validated Docker image.
#
# Do not install `requirements.txt` before this file in a ROCm venv. The CPU
# base pins conflict with the validated ROCm stack and can downgrade it:
#   requirements.txt        ->  this file (ROCm)         reason
#   torch==2.4.1            ->  torch ROCm 7.2.4 wheel   validated gfx1151 kernels
#   (no torchaudio)         ->  torchaudio ROCm 7.2.4     match torch build
#   (no triton-rocm)        ->  triton ROCm 7.2.4         match torch build
#   numpy==2.1.2            ->  numpy==2.4.6              Python 3.12 ROCm image parity
#   scikit-learn==1.5.2    ->  scikit-learn==1.9.0      numpy 2.4 compatibility
#   lightgbm==4.5.0        ->  lightgbm==4.6.0          ROCm overlay pin
# All other pins are identical to requirements-base.txt.
#
# XGBoost upstream GPU support is CUDA-only; no official AMD ROCm XGBoost wheel is
# pinned here. The XGBoost pin stays CPU-safe.
#
# Build:  python3.12 -m venv .venv-rocm
#         .venv-rocm/bin/pip install -r requirements-amd-rocm-full.txt
# Prove:  .venv-rocm/bin/python tools/validate_rocm_acceleration.py --require-gpu
